Primeiro Trabalho Prático

Seja G=(V,E) um grafo simples não-orientado tal que, a cada aresta e ∈ E está associado um custo positivo ce e uma distância de. Dado um caminho P em G, o custo de P é calculado como sendo c(P)=∑e ∈ P ce enquanto o seu comprimento é calculado por d(P)=∑e ∈ P de.

No Problema do Caminho mais Curto com Restrição de Custo (PCMCRC), para o grafo G definido acima, dá-se um inteiro positivo C e dois vértices distintos s e t em V. O objetivo é encontrar o caminho P* de menor comprimento que liga os vértices s e t em G tal que c(P*) ≤ C.

O objetivo deste trabalho é usar a técnica de Relaxação Lagrangeana para produzir bons limitantes duais e, quando possível, limitantes primais para o PCMCRC.

O trabalho será composto de uma parte envolvendo programação e testes computacionais e uma outra parte onde será preparado um relatório reportando os resultados obtidos e respondendo a algumas questões listadas a seguir.

Questões que devem estar respondidas no relatório:

  1. Descreva uma formulação de Programação Linear Inteira (PLI) para o PCMCRC contendo as restrições do problema clássico do Caminho mais Curto (PCMC) e mais uma restrição do tipo mochila binária (KNAP). Explique o significado de todas variáveis e de cada uma das restrições do seu modelo.
  2. Existem pelo menos duas formas de aplicar a Relaxação Lagrangeana para o PCMCRC. Na relaxação RL1 dualiza-se as restrições do PCMC enquanto na relaxação RL2 dualiza-se a restrição KNAP.
    Para qual delas o limitante dual teórico dado pelo Problema Dual Lagrangeano é mais forte ? Explique cuidadosamente a sua resposta.
  3. Qual o problema Problema Primal Lagrangeano a ser resolvido para a relaxação RL1 ? E para a relaxação RL2 ?
  4. Escreva as fórmulas que expressam o custo Lagrangeano das variáveis tanto para RL1 quanto para RL2.
  5. Algoritmos para o Problema Primal Lagrangeano podem ser encontrados em bons livros texto de disciplinas de projeto e análise de algoritmos para as duas relaxações. Diga qual o algoritmo que você usou para resolver o Problema Primal Lagrangeano em cada caso, destacando a sua complexidade de pior caso, se ele é polinomial ou não e citando a fonte das suas informações (referências bibliográficas).
Sobre a implementação e alguns resultados a serem reportados/analisados:
  1. Implemente o método do subgradiente (SM) para resolver o Problema Dual Lagrangeano de RL1 e de RL2.
  2. Execute o SM implementado nos dois casos anteriores para as instâncias de teste. Os formatos dos arquivos para teste são auto-explicativos (abra-os e leia-os)
  3. Exiba tabelas mostrando os limitantes duais obtidos pelos dois métodos para todas as instâncias. Faça uma análise comparativa dos limitantes duais obtidos pelos dois métodos e diga se os resultados obtidos estão de acordo com o que se esperava da teoria.
  4. Sempre que você resolver o Problema Primal Lagrangeano o seu programa deverá verificar se a solução ótima deste problema relaxado é viável para o PCMCRC. Se for, o programa deverá computar o custo do caminho correspondente e gerar um limitante primal. Armazene e reporte o valor do melhor limitante encontrado com este método para todas as instâncias de teste ou diga que não foi possível encontrá-lo. Caso a solução seja viável para o PCMCRC, o programa deve verificar ainda se a solução é ótima. Escreva no seu relatório como este teste foi implementado (ou seja, qual o resultado teórico que permite verificar esta condição) e reporte, se houver, todas as instâncias para as quais você conseguiu comprovar a otimalidade, incluindo o valor ótimo. Note que as situações acima referem-se a ambas as relaxações a serem testadas.
  5. Você deve colocar tabelas no relatório reportando os tempos computacionais totais e os de obtenção de cada um dos melhores limitantes duais e primais (quando houver) para todas as instâncias. Faça uma análise comparativa destes tempos.
  6. Reporte as características da máquina onde os experimentos foram realizados (clock, RAM, processador, etc).
  7. Reporte quais foram os valores dos parâmetros usados no método do subgradiente e outros que alterem o comportamento do algoritmo (tamanho do passo, taxa de redução do tamanho do passo, número de iterações, etc). Considere realizar testes preliminares que mostrem como o comportamento do método do subgradiente é afetado por estes parâmetros (para as duas relaxações !)
  8. Coloque o gráfico de pelo menos uma instância mostrando a variação dos limitantes inferior e superior (se tiver) em função do número de iterações do método do subgradiente. Procure interpretar o gráfico, especialmente aquele referente ao limitante dual, em função das mudanças que ocorreram em parâmetros durante a execução do método (exemplo: alteração no tamanho do passo, melhora do limite primal, etc).
  9. Os nome das instâncias de entrada (veja mais adiante nesta página como baixá-las da rede) são da forma vXX_dYY.dat, onde XX é o número de vértices do grafo de entrada e YY é a sua densidade. Analise o comportamento do SM das duas relaxações em função do aumento do número de vértices e da densidade.

Note que não está sendo solicitada a implementação do algoritmo de branch-and-bound mas, apenas a parte do método do subgradiente !
